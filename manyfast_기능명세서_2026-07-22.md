## 1. 프로젝트 개요
---
### 한 줄 정의
그룹 기업현황 원페이지 대시보드

### 제품 목표
그룹 상장 5개사의 핵심 현황(기본정보·주가·실적·소식)을 필요할 때 30초 내에 확인할 수 있는 단일 원페이지 대시보드를 제공한다.

### 배경
5개사 관련 정보가 증권 앱, 공시 사이트, 뉴스 검색 등 여러 채널에 분산되어 있어 매번 여러 창을 오가며 확인해야 한다. 정기 보고서가 아니라 개인의 수시 확인 행위를 대체하는 ‘북마크형’ 도구가 필요하다.

### 사용자 문제
업무 중 5개사 현황을 짧은 시간에 확인해야 하나 정보가 흩어져 있어 탐색 비용이 크고, 화면을 오가며 핵심을 종합하기 어렵다. 또한 보고서형 산출물은 갱신 주기·형식 제약으로 즉시성이 떨어진다.

### 해결 방안
1920×1080 기준 세로 스크롤 없이 P1(5개사 카드), P2(주가 추이/실적 비교 시각화), P3(기업 소식 리스트)를 한 화면에 배치한다. 주가·실적은 표/텍스트 나열 없이 그래프로 제시하고, 데이터 수집이 불가한 영역은 더미데이터와 ‘API 연동 필요’ 라벨로 명확히 구분해 단계적으로 실데이터로 교체한다.

### 차별점
5개사 고정 구성과 ‘스크롤 0’ 제약을 최우선으로 정보 밀도를 극대화한다. 인증·온보딩·개인화 없이 즉시 진입해 확인하는 사용 맥락에 맞춰, 보고서/포털이 아닌 개인 북마크형 원페이지에 최적화한다. 또한 P2 영역의 표 금지 등 시각화 규칙을 명문화해 일관된 UX를 보장한다.

### 타겟 사용자
INVENI ITC팀 기획 담당 매니저 1인 사용자. 업무 중 수시로 30초 내 확인 후 이탈하며, PC 웹 브라우저(1920×1080)에서 사용한다. 친절한 가이드보다 정보 밀도와 즉시성이 우선이고, 인증/권한/온보딩/개인화는 요구되지 않는다.

### 사용자 시나리오
1) 사용자는 업무 중 그룹사 주가 흐름이 궁금해 대시보드에 접속한다. 2) 상단 카드에서 5개사 기본정보와 시가총액을 훑는다. 3) 좌하단에서 1M/3M/1Y 토글로 5개사 주가 추이를 비교하고, 이어 매출/영업이익 막대 그래프로 분기 실적을 상대 비교한다. 4) 우측 소식 리스트에서 최신 공시/뉴스 헤드라인을 확인하고 필요 시 원문 링크로 이동한다. 5) 30초 내 확인을 마치고 이탈한다.

### 핵심 지표
- 1920×1080에서 세로 스크롤 0회로 P1/P2/P3 전체 노출 달성률(100% 목표)
- 초기 로딩 3초 이내 달성률
- 진입 후 30초 이내 핵심 확인 완료(자기평가 체크/간단한 사용 로그 기반)
- 외부 사이트 방문 없이 대시보드 내에서 확인 완료 비율

### 리스크
- 외부 공개 데이터 수집/정합성 문제(주가·실적·소식 소스별 갱신 지연/불일치)
- KRX OpenAPI 키 발급/쿼터/장애로 인한 주가 데이터 공백
- ‘스크롤 0’ 제약으로 정보 추가 요구 발생 시 범위 충돌(항목 제거/압축 의사결정 필요)
- 더미데이터 영역이 실데이터로 오인될 위험(시각적 구분/라벨 미흡 시)
- 외부망 전용 제약으로 사내 시스템 연동 요구가 생길 가능성

### 카테고리
> 생산성 / 업무

### 사용자 역할
> 기획 매니저(사용자)

### 디바이스 (기기)
> PC 웹(1920×1080)

## 1. 원페이지 레이아웃 및 디자인 시스템
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

사용자가 1920×1080 PC 웹에서 세로 스크롤 없이 5개사 현황을 한 화면에서 확인할 수 있도록, P1/P2/P3 영역 배치와 시각적 규칙을 정의한다. 라이트 비즈니스 톤, 지정 컬러/타이포, 정보 밀도 우선 원칙을 적용하며 불필요한 애니메이션과 트레이딩 터미널 톤을 배제한다. 본 요구사항은 정보 구조와 UI 규칙을 포함하되, 데이터 수집/가공 로직 자체는 포함하지 않는다.

**수용 기준:**
```
1. ○ 1920×1080 기준 P1/P2/P3가 한 화면에 모두 노출되며 세로 스크롤이 발생하지 않는다.
2. ○ P2 영역은 표/텍스트 나열 없이 그래프 중심으로 구성되고 수치는 보조 레이블로만 노출된다.
3. ○ 지정 톤/컬러/타이포(숫자 tabular figures 포함)와 지양 요소(과도한 애니메이션, 3D/그림자 등)가 준수된다.
```


### 1.1 P1/P2/P3 원페이지 고정 배치
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

사용자는 원페이지에서 상단 P1, 좌하 P2, 우측 P3가 동시에 보이는 레이아웃으로 정보를 확인한다. 1920×1080 기준 세로 스크롤이 생기지 않도록 각 영역의 높이/그리드/간격 규칙을 고정한다. 반응형 확장이나 모바일 레이아웃 최적화는 범위에서 제외한다.


#### 1.1.1 스크롤 0 검증 레이아웃 프리셋

사용자는 기본 화면(1920×1080)에서 진입 즉시 P1/P2/P3 전 영역을 스크롤 없이 확인한다. 시스템은 해당 해상도에서 레이아웃이 넘치지 않도록 컴포넌트 높이/여백을 프리셋으로 적용한다. 브라우저 줌/비표준 해상도에서의 완전한 무스크롤 보장은 범위에서 제외한다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)


### 1.2 시각 스타일 가이드 적용
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

라이트 비즈니스 톤을 위해 배경(화이트/라이트 그레이), 텍스트(다크 그레이), 등락 강조색(상승 레드/하락 블루)만 사용하는 스타일을 적용한다. 폰트는 Pretendard 또는 Noto Sans KR를 사용하고 숫자는 tabular figures로 정렬감을 유지한다. 이모지, 과도한 애니메이션, 3D/그림자 등은 사용하지 않는다.


#### 1.2.1 색상

전반적으로 네이비 톤으로 구성하고, 파스텔톤 적극 활용

자극적인 원색은 사용 자제
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

## 2. 계열사 고정 카드 현황(P1)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

5개사(LS일렉트릭, LS에코에너지, LS머터리얼즈, E1, 인베니/예스코홀딩스)를 고정 대상으로 하여 회사명, 주력 사업, 시가총액, 종목코드를 카드 그리드로 제공한다. 사용자는 상단 영역에서 즉시 훑어보며 비교할 수 있어야 한다. 대상 추가/제외 및 사용자별 커스터마이징은 범위에서 제외한다.

**수용 기준:**
```
1. ○ 5개사 고정 카드가 그리드로 표시되며 각 카드에 회사명/주력사업/시가총액/종목코드가 포함된다.
2. ○ 대상 회사의 추가/제외 UI 또는 설정 기능이 제공되지 않는다.
```


### 2.1 5개사 카드 그리드 표시
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

사용자는 상단 카드 그리드에서 5개사 정보를 한눈에 비교한다. 각 카드는 회사명, 주력 사업, 시가총액, 종목코드를 동일한 정보 구조로 표시한다. 카드 순서는 고정이며 사용자 재정렬/숨김은 제공하지 않는다.


#### 2.1.1 카드 모양

카드는 각진 직사각형 모양이 되도록 하고, 갯수가 홀수인 점을 감안해서  배치
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

## 3. 주가 및 실적 시각화(P2)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

5개사의 주가 추이를 동일 그래프(꺾은선)에서 동시 비교하고, 1M/3M/1Y 기간 토글을 제공한다. 실적은 매출과 영업이익을 막대 그래프로 비교해 분기 단위 갱신 데이터를 보여준다. 본 요구사항은 시각화 중심이며, 표 형식 출력 및 과도한 상세 지표 확장은 포함하지 않는다.

**수용 기준:**
```
1. ○ 5개사 주가가 하나의 꺾은선 그래프에서 동시 비교되며 1M/3M/1Y 토글이 동작한다.
2. ○ 실적은 매출/영업이익 막대 그래프로 제공되고 분기 단위 갱신 기준이 명시된다.
3. ○ P2 영역에 표 형태의 데이터 표시가 존재하지 않는다.
```


### 3.1 주가 추이 꺾은선 그래프(5개사 동시)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

사용자는 하나의 꺾은선 그래프에서 5개사 주가 추이를 동시에 비교한다. 기간은 1M/3M/1Y 토글로 전환하며, 그래프 내 수치는 보조 레이블 수준에서만 제공한다. 표 형태의 상세 시세(OHLC, 거래량 등) 제공은 범위에서 제외한다.


#### 3.1.1 기간 토글(1M/3M/1Y) 전환

사용자는 1M/3M/1Y 토글을 클릭해 동일한 그래프에서 기간별 주가 추이를 즉시 전환한다. 시스템은 선택된 기간 상태를 화면 내에서 유지하고, 전환 시 표 형태의 수치 목록을 노출하지 않는다. 기간 커스터마이징(사용자 임의 날짜 선택)은 제공하지 않는다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)


### 3.2 실적 비교 막대 그래프(매출/영업이익)
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

사용자는 분기 기준 매출과 영업이익을 막대 그래프로 비교해 5개사 실적의 상대적 크기를 파악한다. 동일 축/단위 규칙을 적용하고, 수치는 그래프 보조 레이블로만 노출한다. 추가 재무지표(순이익, EPS 등) 확장은 범위에서 제외한다.


#### 3.2.1 막대 

매출, 영업이익 각각 5개사 비교가 가능하도록 구성

사별로 색상이 다르고,  동일 회사의 매출, 영업이익 막대의 색깔은 같다
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

## 4. 기업 소식 집계 및 리스트(P3)
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중

공시/뉴스 등 외부 공개 데이터에서 5개사 관련 소식을 수집해 최신순으로 노출한다. 각 항목은 계열사 태그, 헤드라인, 출처, 일자, 원문 링크를 포함하며 사용자는 필요한 경우 원문으로 이동한다. 알림/푸시, 키워드 편집, 사용자별 필터 저장은 범위에서 제외한다.

**수용 기준:**
```
1. ○ 소식 리스트가 최신순으로 정렬되어 표시된다.
2. ○ 각 소식 항목에 계열사 태그/헤드라인/출처/일자/원문 링크가 포함된다.
```


### 4.1 기업 소식 최신순 리스트
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

사용자는 우측 리스트에서 5개사 관련 소식을 최신순으로 확인한다. 각 항목에 계열사 태그, 헤드라인, 출처, 일자, 원문 링크를 표시하며, 링크 클릭 시 새 탭/창으로 원문으로 이동한다. 사용자 맞춤 필터 저장이나 알림은 제공하지 않는다.


#### 4.1.1 원문 링크 아웃바운드 이동

사용자는 소식 항목의 원문 링크를 클릭해 공시/뉴스 원문으로 이동한다. 시스템은 대시보드 이탈로 인한 맥락 손실을 줄이기 위해 새 탭/창으로 링크를 연다. 링크 추적/북마크/읽음 처리 기능은 범위에서 제외한다.
> **중요도**: 🟡 중간 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

## 5. 데이터 소스 연동 및 더미데이터 표기
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중

주가는 KRX OpenAPI를 통해 연동하고, 실적/계열사 현황/소식은 외부 공개 데이터 수집으로 채운다. 수집이 불가하거나 미연동인 영역은 더미데이터를 사용하되 ‘API 연동 필요’ 라벨을 필수로 표시하고 실데이터와 시각적으로 구분한다. 본 요구사항은 외부망 전용 제약을 전제로 하며 사내 시스템 연동은 포함하지 않는다.

**수용 기준:**
```
1. ○ 주가 데이터는 KRX OpenAPI 연동 전제로 설계되며 미연동 시 더미데이터와 라벨로 구분된다.
2. ○ 실적/계열사 현황/소식 중 미수집 영역은 더미데이터 + ‘API 연동 필요’ 라벨이 표시된다.
3. ○ 사내 시스템 연동 기능이나 내부망 의존 요소가 없다(외부망 전용).
```


### 5.1 KRX OpenAPI 주가 데이터 연동
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

시스템은 KRX OpenAPI를 통해 5개사 종목의 주가 시계열 데이터를 가져와 주가 추이 그래프에 반영한다. API 키/쿼터/장애 등으로 데이터를 가져오지 못하는 경우 더미데이터로 대체하고 ‘API 연동 필요’ 또는 데이터 공백 안내를 표시한다. 사내망/내부 시스템 호출은 수행하지 않는다.


### 5.2 미연동 영역 더미데이터 및 라벨 표기
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)

실적/계열사 현황/소식 등 수집 불가 또는 미완료 데이터는 더미데이터로 채우되 실데이터와 시각적으로 구분한다. 해당 컴포넌트에는 ‘API 연동 필요’ 라벨을 항상 노출해 오인 가능성을 줄인다. 연동 완료 시 더미데이터를 실데이터로 순차 교체할 수 있어야 한다.


#### 5.2.1 더미데이터 시각적 구분 및 라벨 상시 노출

사용자는 더미데이터가 포함된 카드/그래프/리스트에서 ‘API 연동 필요’ 라벨을 통해 실제 연동 여부를 즉시 구분한다. 시스템은 더미데이터 영역을 실데이터와 다른 스타일(예: 톤 다운, 워터마크 등)로 표시해 오인을 방지한다. 라벨을 임의로 숨기거나 사용자가 해제하는 옵션은 제공하지 않는다.
> **중요도**: 🔴 높음 | **진행 상태**: ⚪ 작성중 | **역할**: 기획 매니저(사용자) | **기기**: PC 웹(1920×1080)
