# PRD — 그룹 통합 기업현황 원페이지 대시보드

v0.3 | 2026-07-22 | INVENI ITC팀 기획 매니저

| 구분 | 세부 항목 | 내용 |
|---|---|---|
| **1. 목적** | 배경 | 5개사 현황이 증권 앱·공시 사이트·뉴스 검색으로 흩어져 있어 매번 여러 창을 오가야 함 |
| | 목적 | 그룹 상장 5개사 현황을 궁금할 때 즉시 확인하는 상시 조회용 단일 화면 제공 |
| | 성격 | 정기 보고·배포용 리포트가 아닌, 개인의 일상적 확인 행위를 대체하는 도구 ("보고서"가 아니라 "북마크") |
| **2. 타깃** | 사용자 | ITC팀 기획 담당 매니저 본인 (1인 사용) |
| | 사용 맥락 | 업무 중 수시로, 짧게 — 30초 이내 확인 후 이탈 |
| | 환경 | PC 웹 브라우저, 1920×1080 기준 |
| | 설계 함의 | 인증·권한·온보딩·개인화 불필요 / 친절함보다 정보 밀도 우선 |
| **3. 핵심 기능** | 대상 계열사 (고정) | LS일렉트릭, LS에코에너지, LS머터리얼즈, E1, 인베니(예스코홀딩스) — 5개사 고정, 추가·제외 미제공 |
| | P1 계열사 현황 | 회사명·주력 사업·시가총액·종목코드 → 5개사 카드 그리드 |
| | P2 주가 / 실적 | 주가 추이 **꺾은선**(5개사 동시, 1M/3M/1Y 토글) + 실적 비교 **막대**(매출·영업이익) |
| | P3 기업 소식 | 계열사 태그·헤드라인·출처·일자·원문 링크 → 최신순 리스트 |
| | 시각화 규칙 | P2 영역의 텍스트 나열·표 형식 **금지**. 수치는 그래프 보조 레이블로만 노출 |
| **4. 화면** | 구성 | 원페이지 1장에 P1 / P2 / P3 배치 (상단 P1, 좌하 P2, 우측 P3) |
| | 최상위 제약 | 1920×1080 기준 **세로 스크롤 0** — 배치는 이 제약을 우선으로 조정 |
| | 톤 | 라이트 비즈니스형 (깔끔한 사내 보고 톤) |
| | 컬러 | 배경 화이트·라이트 그레이(#FFFFFF, #F7F8FA), 텍스트 다크 그레이, 강조색은 등락 표기(상승 레드·하락 블루)에 한정 |
| | 타이포 | Pretendard 또는 Noto Sans KR / 숫자는 tabular figures |
| | 지양 | 다크 트레이딩 터미널 톤, 과도한 애니메이션, 이모지, 캐주얼 위젯, 3D·그림자 |
| **5. 데이터** | 주가 | **KRX OpenAPI** 연동 (API key·명세서는 개발 단계에서 별도 제공) |
| | 실적 | 공시 등 외부 공개 데이터 수집, 분기 단위 갱신 |
| | 계열사 현황 | 공시 등 외부 공개 데이터 수집 |
| | 기업 소식 | 공시·뉴스 등 외부 공개 데이터 수집 |
| | 수집 불가 영역 | **더미데이터 + "API 연동 필요" 라벨 표기 필수** (실데이터와 시각적 구분, 연동 시점에 순차 교체) |
| | 제약 | **외부망 전용** — 사내 시스템 연동 불가 |
| **6. 성공 기준** | S1 (필수) | 1920×1080에서 **세로 스크롤 0회**로 P1·P2·P3 전체 노출 |
| | S2 | 진입 후 30초 이내 5개사 주가·실적·소식 파악 |
| | S3 | 초기 로딩 3초 이내 |
| | S4 | 별도 사이트 방문 없이 대시보드만으로 확인 완료 |
| | 실패 조건 | 정보를 더 담기 위해 스크롤이 생기는 순간 실패 → 항목 추가 시 기존 항목 제거·압축 전제 |
